# Краткое описание

Компиляция:
```bash
gcc -std=c11 -Wall -Wextra -o copy_sys copy_sys.c
```

Запуск:

Стандартный режим (большой буфер, размер > size файла):
```
./copy_sys file_source.txt file_dest.txt
```

Циклический режим с 32-байтным буфером:
```
./copy_sys -c file_source.bin file_dest.bin
```

Поведение прав доступа:

После создания файл получает те же права r/w и т.п. что и исходный.

Если исходный файл имел любой execute-бит (u/x, g/x или o/x), то они будут установлены и у целевого файла — это обеспечивает запуск скриптов и исполняемых файлов.

Если исходный файл не исполняемый — execute-биты в целевом файле сброшены.

Ограничения и замечания:

Программа не использует стандартные потоковые функции для операций с файлами (fopen/fread/fwrite), только системные вызовы open/read/write/close/fstat/fchmod.

Программа избегает перезаписи самого себя: если source и dest указывают на один и тот же inode — выдаётся ошибка.

В big-buffer режиме выделяется динамический буфер размера st_size + 1. Для очень больших файлов это может потребовать много памяти; если это нежелательно, используйте -c режим.

Поддерживаются как текстовые, так и бинарные файлы.
